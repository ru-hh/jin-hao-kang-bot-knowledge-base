## 複雜個案測試

### 案例：多重複合性脆弱家庭

**家庭狀況**：
- 案主（63歲男性）：高齡勞工、聽力受損、從事高負荷搬運工、收入不穩
- 案次女（34歲女性）：躁鬱症傾向、大腸癌及卵巢癌病史、身上有醫療管路、無法工作
- 急迫危機：面臨**強制驅離**，居住與經濟同時陷入困境
- 家庭支持系統薄弱（離異、長女失聯）

**系統回應表現**：

1. **展現「輕重緩急」判斷力**
   - 首先識別「強制驅離」為最大風險
   - 將緊急救援資源（急難救助金）列為最優先
   - 展現「先安頓、後輔導」的救助邏輯

2. **規劃「短中長期」福利藍圖**
   - 短期（救急）：馬上關懷 + 300億租金補貼
   - 中長期（治本）：建立低收入戶/中低收入戶身分，作為醫療減免基礎

3. **執行「分眾化」資源媒合**
   - 對案主（63歲）：推薦中高齡者就業促進資源
   - 對次女（34歲）：建議身心障礙鑑定，連結醫療與身障資源

4. **提供具體行動路徑**
   - 1.立即申請 → 2.搬遷與租屋 → 3.建立資格
   - 對非社工背景的志工而言＝一份可直接執行的「個案處遇計畫」

**結論**：已具備處理「非結構化自然語言」與「多重問題複合案例」的高階能力，成功模擬資深社工的評估邏輯。

## 負面回饋與決策

### 1. 使用者輸入門檻
- **回饋**：年長志工希望多輪對話
- **決策**：維持「單輪精準查詢」
- **理由**：資安考量（阻斷攻擊鏈）、系統定位（目的導向非閒聊）、Gemini 3 可處理長篇幅單次提問

### 2. 資訊分級
- **回饋**：資訊量過大，一般民眾希望快速看結論
- **決策**：導入「一句話總結」分層回應
- **設計**：頂層【一句話總結】+ 下層【詳細解析與依據】

### 3. 資源範疇
- **回饋**：志工希望納入慈濟、家扶等 NGO 資源
- **決策**：現階段不納入（依 114.12.23 會議決議）
- **理由**：維護政府資訊權威性、民間資源變動頻繁恐致幻覺
